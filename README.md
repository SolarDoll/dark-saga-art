# Dark Saga Art

Персональный сайт-портфолио мастера авторских кукол и чайных игрушек (tea pets / ти-петов).

## О проекте
Данный репозиторий представляет собой персональную страницу мастера. Здесь будут представлены уникальные работы автора:
- **Авторские художественные куклы.**
- **Ти-петы (чайные фигурки)** — глиняные фигурки для чайных церемоний.

Визуальный стиль, дизайн и финальная структура сайта находятся в процессе исследования и проектирования. Разработка ведется итеративно.

## Известные сложности и особенности (Pitfalls)
В процессе разработки мы фиксируем ключевые нюансы, технические проблемы и архитектурные решения здесь, чтобы будущие агенты не совершали ошибок.

### Окружение и Инструменты
1. **GitHub CLI (`gh`) и PATH в Windows:** После установки утилиты `gh` через менеджер пакетов `winget`, переменные окружения `PATH` не обновляются в текущей сессии терминала автоматически. Требуется либо перезапуск оболочки, либо принудительное обновление переменной пути:
   ```powershell
   $env:Path = [System.Environment]::GetEnvironmentVariable("Path", "Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path", "User")
   ```
2. **Конфликт флага `-f` в PowerShell:** При вызове команд GitHub CLI (например, `gh api`) короткий флаг `-f` (поле запроса) воспринимается PowerShell как оператор форматирования строк, что приводит к синтаксическим ошибкам. **Решение:** всегда использовать длинные флаги `--raw-field` или `--field`.
3. **Первое SSH-соединение с GitHub:** При первом клонировании или проверке соединения через SSH, Git ожидает подтверждения отпечатка ключа (fingerprint). Для автоматизации проверок в скриптах используйте опцию `-o StrictHostKeyChecking=accept-new`:
   ```bash
   ssh -T -o StrictHostKeyChecking=accept-new git@github.com
   ```
